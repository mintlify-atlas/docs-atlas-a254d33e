NexPhisher ships with 30 platform entries covering social media, email, gaming, finance, and other services. Four platforms — Facebook, Instagram, Google, and VK — expose a sub-menu with multiple page variants.

## File locations

All template files are stored under `.Modules/` in the repository root. Each platform has its own subdirectory:

```text
.Modules/
├── facebook/
├── fb_advanced/
├── fb_security/
├── fb_messenger/
├── instagram/
├── ig_followers/
├── ig_verify/
├── google/
├── google_new/
├── google_poll/
├── microsoft/
├── netflix/
├── ...
```

When you select a template, NexPhisher copies the matching subdirectory's contents into `.htr/www/`, which is the document root served by the PHP web server.

## Log files

Captured data is written to the `logs/` directory:

| File | Contents |
|------|----------|
| `logs/<website>.log` | Captured credentials (username/password pairs). `<website>` matches the internal variable name listed in the tables below. |
| `ip.txt` | IP addresses of visitors who loaded the phishing page. |

For example, credentials captured by the `facebook` template are written to `logs/facebook.log`.
