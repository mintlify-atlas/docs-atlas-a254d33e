---
title: "Quickstart"
description: "Get from a fresh install to a running phishing page in a few steps."
---

<Warning>
  Only use NexPhisher on systems and accounts you own or have explicit written permission to test. Unauthorized use is illegal.
</Warning>

<Steps>
  <Step title="Clone and set up">
    If you haven't already completed installation, clone the repository and run the setup script:

    ```bash
    apt update ; apt install git -y ; git clone git://github.com/htr-tech/nexphisher.git ; cd nexphisher ; bash setup ; bash nexphisher
    ```

    On Termux, replace `bash setup` with `bash tmux_setup`.

    See the [Installation](/installation) page for a full breakdown of what setup does.
  </Step>

  <Step title="Launch NexPhisher">
    From inside the `nexphisher/` directory, run:

    ```bash
    bash nexphisher
    ```

    NexPhisher checks that all requirements are present (`php`, `curl`, `ssh`, `unzip`, `.htr/`, `.Modules/`) and then displays the interactive menu.
  </Step>

  <Step title="Select a phishing template">
    The main menu lists 30 platforms numbered `01` through `30`. Enter the number for the platform you want to simulate.

    ```
    [01] Facebook        [11] Twitch          [21] Snapchat
    [02] Instagram       [12] Pinterest       [22] Microsoft
    [03] Google          [13] Tumblr          [23] Steam
    [04] Microsoft       [14] Reddit          [24] PayPal
    ...
    [99] About           [00] Exit
    ```

    For platforms like **Facebook**, **Instagram**, **Google**, and **VK**, a sub-menu appears with variant options (for example, classic login, two-factor prompt, or mobile layout). Select the variant you want.
  </Step>

  <Step title="Choose a tunneling option">
    After selecting a template, NexPhisher presents the tunnel menu:

    ```
    [01] Localhost
    [02] Ngrok
    [03] LocalXpose
    [04] LocalHostRun
    [05] Custom
    ```

    Select a tunnel to expose the locally-served phishing page over the internet.

    <Note>
      NexPhisher serves phishing pages on `127.0.0.1` using PHP's built-in server:

      - **Localhost / Ngrok**: port `5555`
      - **LocalXpose**: port `4142`
      - **LocalHostRun**: port `4545`
    </Note>
  </Step>

  <Step title="Share the generated link">
    Once the tunnel is established, NexPhisher prints a public URL. This is the link you share as part of your authorized test. When a target visits the URL and submits the form, their credentials are captured automatically.
  </Step>

  <Step title="Review captured credentials">
    Captured credentials and IP addresses are written to files inside the `logs/` directory as soon as a form submission is received.

    **Credential log** (username and password):

    ```
    logs/<website>.log
    ```

    For example, a Facebook template writes to `logs/facebook.log`.

    **IP address log**:

    ```
    logs/ip.txt
    ```

    You can watch the log file in real time:

    ```bash
    tail -f logs/<website>.log
    ```
  </Step>
</Steps>
