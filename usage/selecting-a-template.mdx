## Platforms with sub-menus

Four platforms present a secondary menu with multiple page variants. Select the variant that best fits your testing scenario.

<AccordionGroup>
  <Accordion title="Facebook (01)">
    ```text
    [01] Traditional Login
    [02] Advanced Voting Poll
    [03] Fake Security Login
    [04] Facebook Messenger
    ```

    | Option | Description |
    |---|---|
    | 01 | Standard Facebook login page |
    | 02 | A fake voting poll that redirects to a login prompt |
    | 03 | A fake account security warning with a login form |
    | 04 | Facebook Messenger login page |
  </Accordion>
  <Accordion title="Instagram (02)">
    ```text
    [01] Traditional Login
    [02] Auto Followers
    [03] Blue Badge Verify
    ```

    | Option | Description |
    |---|---|
    | 01 | Standard Instagram login page |
    | 02 | Fake "auto followers" service that requires login |
    | 03 | Fake verified badge application that requires login |
  </Accordion>
  <Accordion title="Google (03)">
    ```text

    [03] Advanced Voting Poll
    ```

    | Option | Description |
    |---|---|

    | 03 | A fake Google voting poll that redirects to a login prompt |
  </Accordion>

  </Accordion>

## After selecting a template

Once you confirm a selection (and a sub-menu option, if applicable), NexPhisher copies the corresponding template files from `.Modules/` into `.htr/www/`. The tunnel selection menu is then displayed. See [Port forwarding](/usage/port-forwarding).
